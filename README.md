## sutils monitor

Auto-shutdown your server when all monitored ports have been idle for a configured period. Keeps costs down by turning off the machine when no one is using it.

### How it works

1. Reads a `config.yaml` file with a list of ports to watch
2. Checks every `check_interval` seconds for established TCP connections on those ports
3. Once **all ports** have had zero active connections for the full `grace_period`, it runs your configured shutdown command
4. A new connection during the grace period resets the timer

### Requirements

- Linux (uses `ss` or falls back to `netstat`)
- Node.js >= 16

### Quick start

```bash
# 1. Create a config file
sudo mkdir -p /etc/sutils
npx @vantran-se/sutils monitor init
sudo mv config.yaml /etc/sutils/config.yaml

# 2. Edit the config
sudo nano /etc/sutils/config.yaml

# 3. Install and start as a systemd service

sutils monitor enable --config /etc/sutils/config.yaml
sutils monitor start
```

### Configuration

`config.yaml` (generated by `init`):

```yaml
# How often to check for active connections (seconds)
check_interval: 60

# How long all ports must be idle before shutdown is triggered (seconds)
grace_period: 900

# Command to run when shutting down
shutdown_command: "sudo shutdown -h now"

# Ports to monitor — shutdown only triggers when ALL of these are idle
ports:
  - 22    # SSH – someone is logged in
  - 3000  # your app port
```

| Field | Description |
|---|---|
| `check_interval` | Seconds between each connection check |
| `grace_period` | Seconds all ports must be idle before shutdown is triggered |
| `shutdown_command` | Shell command to execute when shutting down |
| `ports` | List of TCP ports to monitor |

### CLI reference

```
sutils monitor <subcommand> [options]

Subcommands:
  init      Create a starter config.yaml in the current directory
  run       Run the monitor process directly in the foreground

  enable    Install and enable the systemd service
  disable   Stop and remove the service from boot
  start     Enable on boot and start the service now
  stop      Stop the service (stays enabled, restarts on reboot)
  status    Show systemd service status

Options:
  --config <path>   Path to config file (default: ./config.yaml)
  --dry-run         Log what would happen without executing shutdown (run only)
```

### Systemd service workflow

```bash
# Install the service and start it (enable + start in one step)
sutils monitor enable --config /etc/sutils/config.yaml
sutils monitor start

# Day-to-day controls
sutils monitor stop      # stop now, restarts on next reboot
sutils monitor start     # start again (re-enables on boot too)
sutils monitor status                        # status doesn't need root
sutils monitor disable   # remove from boot
```

> `start` always enables on boot. To stop temporarily without affecting boot behaviour, use `stop`.

### Running the monitor directly (without systemd)

```bash
# Useful for testing or non-systemd environments
sutils monitor run --config ./config.yaml

# Safe test — logs what would happen without shutting down
sutils monitor run --dry-run
```

### Permissions for shutdown

The service needs permission to run the shutdown command. Either run as root, or grant passwordless sudo for shutdown:

```bash
# /etc/sudoers.d/sutils
your-user ALL=(ALL) NOPASSWD: /sbin/shutdown
```
