# sutils

Server utilities CLI — a collection of tools for managing and maintaining servers.

## Commands

| Command | Description |
|---|---|
| `sutils monitor` | Monitor port connections and auto-shutdown when idle |

---

## sutils monitor

Auto-shutdown your server when all monitored ports have been idle for a configured period. Keeps costs down by turning off the machine when no one is using it.

### How it works

1. Reads a `config.yaml` file with a list of ports to watch
2. Checks every `check_interval` seconds for established TCP connections on those ports
3. Once **all ports** have had zero active connections for the full `grace_period`, it runs your configured shutdown command
4. A new connection during the grace period resets the timer

### Requirements

- Linux (uses `ss` or falls back to `netstat`)
- Node.js >= 16

### Quick start

```bash
# Create a config file in your current directory
npx sutils monitor init

# Edit the config
nano config.yaml

# Start monitoring
npx sutils monitor start
```

### Configuration

`config.yaml` (generated by `init`):

```yaml
# How often to check for active connections (seconds)
check_interval: 60

# How long all ports must be idle before shutdown is triggered (seconds)
grace_period: 300

# Command to run when shutting down
shutdown_command: "sudo shutdown -h now"

# Ports to monitor — shutdown only triggers when ALL of these are idle
ports:
  - 22    # SSH – someone is logged in
  - 3000  # your app port
```

| Field | Description |
|---|---|
| `check_interval` | Seconds between each connection check |
| `grace_period` | Seconds all ports must be idle before shutdown is triggered |
| `shutdown_command` | Shell command to execute when shutting down |
| `ports` | List of TCP ports to monitor |

### CLI reference

```
sutils monitor <subcommand> [options]

Subcommands:
  init     Create a starter config.yaml in the current directory
  start    Start monitoring

Options:
  --config <path>   Path to config file (default: ./config.yaml)
  --dry-run         Log what would happen without executing shutdown
```

### Examples

```bash
# Use a custom config path
sutils monitor start --config /etc/sutils/config.yaml

# Test your config without risk — won't actually shut down
sutils monitor start --dry-run
```

### Running as a background service

#### systemd (recommended for Linux servers)

Create `/etc/systemd/system/sutils-monitor.service`:

```ini
[Unit]
Description=sutils monitor
After=network.target

[Service]
ExecStart=npx sutils monitor start --config /etc/sutils/config.yaml
Restart=on-failure
User=root

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl enable sutils-monitor
sudo systemctl start sutils-monitor
sudo systemctl status sutils-monitor
```

#### Permissions for shutdown

The process needs permission to run the shutdown command. Either run as root, or grant the user passwordless sudo for shutdown:

```bash
# /etc/sudoers.d/sutils
your-user ALL=(ALL) NOPASSWD: /sbin/shutdown
```
